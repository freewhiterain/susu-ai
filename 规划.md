## 三、核心模块设计

### 3.1 RAG Pipeline

```
上传文档
  │
  ├─ 格式检测（MD / PDF / Word / TXT）
  ├─ 文本提取（pypdf / python-docx / markdown）
  ├─ Chunking（按段落，500 token，50 token 重叠）
  ├─ Embedding（OpenAI text-embedding-3-small）
  └─ 写入 ChromaDB（含 doc_id / filename / chunk_index 元数据）

查询时
  │
  ├─ 查询文本 Embedding
  ├─ ChromaDB 语义检索（Top-5）
  ├─ 返回 chunk 文本 + 来源（文件名 + chunk 索引）
  └─ 组装引用卡片给模型
```

**增量更新**：上传同名文件时，先删除旧向量（按 doc_id 过滤），再重新索引。

### 3.2 Agent 主循环

> **流式说明**：Web 聊天页通过 SSE 逐 token 推送（`stream=True`）；飞书/钉钉 **IM 端不支持流式消息**，必须等完整回复生成后一次性调 IM API 发送。`agent.py` 提供两个入口：`run_stream()` 给 Web SSE，`run()` 给 IM。

```python
# 伪代码（IM 场景，非流式）
async def run(user_msg: str, session_id: str) -> AgentResponse:
    history = await session.get(session_id)
    history.append({"role": "user", "content": user_msg})

    while True:
        response = await llm.chat_with_tools(history, TOOLS, stream=False)

        if response.stop_reason == "end_turn":
            break  # 生成结束，跳出循环

        # 模型要求调工具，执行后把结果追加进 history，再次调模型
        for tool_call in response.tool_calls:
            result = await dispatch_tool(tool_call.name, tool_call.input)
            history.append(tool_result_message(tool_call, result))

    await session.save(session_id, history)
    await log_conversation(session_id, history, response.usage)  # 写 SQLite + Langfuse
    return response

# Web SSE 场景：逐 token yield，工具调用时中断流、执行、再重新流
async def run_stream(user_msg: str, session_id: str) -> AsyncIterator[str]:
    ...  # 同上逻辑，但 llm.chat_with_tools(..., stream=True) 且逐 chunk yield
```

### 3.3 LLM 抽象层（多模型适配）

> **设计修正**：`embed()` 不能放进 `LLMClient`——Claude 本身没有 Embedding API，放进去会导致 `ClaudeClient.embed()` 内部偷偷调 OpenAI，违背抽象层意义。对话客户端和向量化客户端拆成两个独立抽象。

```
LLMClient (ABC)
  └── chat_with_tools(messages, tools, stream) → AsyncIterator[LLMChunk]

ClaudeClient(LLMClient)      # anthropic SDK，Tool Use API，支持流式
OpenAIClient(LLMClient)      # openai SDK，function_calling，支持流式

EmbeddingClient (ABC)
  └── embed(text) → list[float]

OpenAIEmbeddingClient(EmbeddingClient)   # text-embedding-3-small
VoyageEmbeddingClient(EmbeddingClient)   # voyage-3，Anthropic 官方推荐搭 Claude 使用
LocalEmbeddingClient(EmbeddingClient)    # sentence-transformers，无需外部 Key
```

三种 Embedding 选项对比：
| 选项 | Key 要求 | 适用场景 |
|------|---------|---------|
| OpenAI text-embedding-3-small | OPENAI_API_KEY | 默认，生态成熟 |
| Voyage AI voyage-3 | VOYAGE_API_KEY | 搭 Claude 时 Anthropic 官方推荐 |
| sentence-transformers（本地） | 无需 Key | 离线/无预算场景 |

环境变量：
- `LLM_PROVIDER=claude|openai` 控制对话模型
- `EMBEDDING_PROVIDER=openai|voyage|local` 控制向量化（与 LLM 解耦，独立配置）
- `ANTHROPIC_API_KEY` 仅 `LLM_PROVIDER=claude` 时必填

### 3.4 IM 适配层

```
IMAdapter (ABC)
  ├── verify_signature(request) → bool
  ├── parse_event(body) → IMEvent
  └── send_message(chat_id, content, references) → None

DingTalkAdapter(IMAdapter)   # 主
FeishuAdapter(IMAdapter)     # 加分项
```

会话上下文 Redis Key：`session:{platform}:{user_id}:{chat_id}`

### 3.5 Langfuse 可观测性

在 `agent.py` 的 `run()` 函数加 `@observe()` 装饰器，自动追踪：
- 完整 prompt / response
- 每次工具调用的入参和结果
- Token 用量（input / output / total）
- 延迟时间

### 3.6 MCP Server

暴露 3 个工具供 Claude Desktop / Cursor 调用：
- `search_docs(query)` → 知识库检索结果
- `call_employee(emp_id)` → 员工信息
- `get_attendance(emp_id, date_range)` → 考勤记录

### 3.7 Token 成本展示

每次对话写入 `ConversationLog`：
```
input_tokens: int
output_tokens: int
cost_usd: float   # input * INPUT_PRICE_PER_M / 1_000_000 + output * OUTPUT_PRICE_PER_M / 1_000_000
```

定价走 env，不硬编码，切模型时只改配置：
```bash
INPUT_PRICE_PER_M=3.0    # claude-sonnet-4-6 默认，切 gpt-4o 时改为 2.5
OUTPUT_PRICE_PER_M=15.0  # claude-sonnet-4-6 默认，切 gpt-4o 时改为 10.0
```
前端日志表展示，仪表盘累计今日成本。

### 3.8 API 重试与限流处理

`claude.py` / `openai_client.py` 用 `tenacity` 装饰器处理瞬时错误，工程化硬要求：

```python
from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception

@retry(
    retry=retry_if_exception(lambda e: getattr(e, "status_code", 0) in {429, 503}),
    wait=wait_exponential(multiplier=1, min=2, max=30),
    stop=stop_after_attempt(3),
)
async def _call_llm_api(...): ...
```

- `429 Rate Limit`：指数退避重试，最多 3 次
- `503 / 网络超时`：同上
- 3 次仍失败：抛自定义 `LLMUnavailableError`，上层 agent 捕获后返回兜底回复，IM 端不能无响应

## 七、Docker Compose 服务清单

> **ChromaDB 设计修正**：ChromaDB 支持 `PersistentClient(path="./data/chroma")` 内嵌在 Python 进程里，无需单独容器。去掉独立的 chromadb 服务，减少运维复杂度，数据目录挂载到 backend 容器的 volume 即可。

```yaml
services:
  backend:      # FastAPI，port 8000（内嵌 ChromaDB，挂载 ./data/chroma）
  frontend:     # Next.js，port 3000
  mock-api:     # FastAPI Mock，port 8001
  redis:        # Redis 7，port 6379
```

启动：`docker-compose up --build`
停止：`docker-compose down`
