## 二、技术亮点

1. **抽象分层干净**：`LLMClient` 与 `EmbeddingClient` 分离（Claude 无 embedding API），便于替换模型与向量源（OpenAI/Voyage/本地三选一）。
2. **流式与 IM 双入口**：`run_stream()` 服务 Web（SSE），`run()` 服务 IM（完整响应后经 sessionWebhook 回发），共享同一工具循环。
3. **健壮降级**：Redis 不可用降级为无记忆；Langfuse 未配置静默跳过；LLM 异常返回友好兜底；工具超时不阻断主流程。
4. **安全合规**：密钥仅经 `.env` 注入，源码零硬编码，`.gitignore` 严格排除；设置接口绝不下发任何 key（有测试守护）。
5. **成本可控**：Token 单价配置化，避免硬编码价格随模型变动而失真。

## 三、测试情况

- **单元/集成测试**：RAG 切片与检索、文档增删改查、Agent 工具选择与多轮上下文、内部 API 工具（含超时）、钉钉验签与消息解析、设置接口（含密钥不泄露断言）、MCP 工具暴露。
- **离线评测**：`evals/run_evals.py` 对 20 用例打分，输出各维度通过率与 JSON 报告。

运行：`./scripts/test.sh` 与 `uv run python -m evals.run_evals`。

## 四、已知限制与改进方向

| 限制 | 说明 / 后续 |
| --- | --- |
| 检索为朴素向量召回 | 可加入 BM25 混合检索、Rerank 提升精度 |
| 后端单进程 | ChromaDB 内嵌不适合多 worker；规模化可换独立向量服务 |
| 引用结构较简 | 当前以文本片段返回，可结构化为 {文件, 页码, 分数} 并支持点击定位原文 |
| 评测依赖真实 LLM | 评测需联网调用，可补充基于固定 fixture 的离线快照测试 |
| 前端无鉴权 | 内网工具未做登录；上生产需接入 SSO |
